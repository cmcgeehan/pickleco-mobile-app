PRD: Lessons Booking System

Objective

Allow players to book private lessons with coaches, factoring in coach availability, membership discounts, and payment via Stripe.

⸻

🧩 Scope & Key Components
	1.	Coach Availability Management (✅ COMPLETE)
	•	Admin or coaches can define their available time slots via API endpoints
	•	Availability limits options during the lesson booking flow
	•	Table: coach_availability with fields: id, coach_id, start_time, end_time, recurring_rule

	2.	Lesson Booking Flow (🔄 IN PROGRESS)
	•	Step 1: Choose a coach (dropdown with "Any available coach" option)
	•	Step 2: Choose a date (limit to coach's available days)
	•	Step 3: Choose a time (only time slots coach is available)
	•	Step 4: Choose a venue/court (can reuse existing court selection flow)
	•	Step 5: Confirmation & Payment (price calculated via coach rate and membership discount)
	
	3.	Pricing Logic (✅ COMPLETE)
	•	Pull users.coaching_rate for coaches
	•	Multiply by lesson duration (default 1 hour)
	•	Apply user's membership discount from membership_event_costs where event_type = 'Lesson'
	
	4.	Data Integration (✅ COMPLETE)
When a lesson is booked, create the following records:
	•	events: one record with event_type_id corresponding to "Lesson", coach_id, lesson_duration
	•	event_registrations: one for each participant (player + added guests if applicable)
	•	event_courts: associate a court_id with the event
