# Membership Checkout Debugging Lessons (2024-07-17)

## Key Lessons and Patterns

1. **Explicit JWT Handling in API Routes**
   - Always extract the JWT from the `authorization` header (or fallback to the `supabase.auth.token` cookie) in API routes that require authentication.
   - Initialize the Supabase client with the JWT using the `global.headers.Authorization` option to ensure all queries run as the authenticated user.
   - This is required because the default cookie-based session handling may not work in all API contexts (see Stripe and membership activation routes).

2. **Debugging Session/Auth Issues**
   - Add debug output to log incoming cookies, headers, and the result of session/user fetches in API routes.
   - Check for the presence and validity of the JWT in both the header and cookie.
   - Compare the session/user ID with the expected user ID to catch mismatches early.

3. **Correct Supabase Client Initialization Pattern**
   - For authenticated API routes, use:
     ```ts
     const supabase = createClient(
       process.env.NEXT_PUBLIC_SUPABASE_URL!,
       process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
       {
         global: {
           headers: {
             Authorization: jwt ? `Bearer ${jwt}` : ''
           }
         }
       }
     );
     ```
   - Use `supabase.auth.getUser()` to fetch the authenticated user.
   - Always check for errors and null results, and return 401 if the session is missing or invalid.

4. **Reference**
   - The full debug log and investigation steps are archived in `implementations/membership-checkout-debug-2024-07-17.txt` for future reference.

# Logout Function Debugging Lessons (2024-12-19)

## Key Lessons and Patterns

1. **Duplicate Auth State Change Listeners**
   - **NEVER have multiple `onAuthStateChange` listeners in the same component**
   - Multiple listeners create race conditions and inconsistent state clearing
   - Always use a single, centralized auth state change listener
   - Check for duplicate `useEffect` hooks that set up auth listeners

2. **Auth State Clearing on Logout**
   - Ensure the `reset()` function clears ALL relevant state properties:
     ```typescript
     reset: () => {
       set({
         user: null,
         profile: null,
         error: null,
         session: null,
         isAdmin: false,
         isCoach: false
       })
     }
     ```
   - Include role flags (`isAdmin`, `isCoach`) in the reset
   - Verify that UI components check the correct state properties

3. **Debugging Auth State Issues**
   - Add debug logging for auth state changes:
     ```typescript
     console.log('Auth state change:', event, session?.user?.id)
     console.log('SIGNED_OUT event - resetting auth state')
     ```
   - Check for duplicate listeners first when auth state persists incorrectly
   - Verify that `reset()` is being called on SIGNED_OUT events

4. **AuthProvider Best Practices**
   - **Single auth state change listener per component**
   - **Proper cleanup in useEffect return function**
   - **Consistent reset() calls on SIGNED_OUT**
   - **Debug logging for auth events**

5. **Code Review Checklist for Auth Issues**
   - [ ] Check for duplicate auth state change listeners
   - [ ] Verify single auth initialization pattern
   - [ ] Ensure proper cleanup in useEffect
   - [ ] Test logout flow end-to-end
   - [ ] Verify reset() function clears all relevant state

6. **Reference**
   - The full debug log and investigation steps are archived in `implementations/logout-function-debug-2024-12-19.md` for future reference.

```typescript
export class SessionManager {
  // Refresh strategy configuration
  private readonly REFRESH_INTERVAL = 4 * 60 * 1000; // 4 minutes
  private readonly MAX_RETRY_ATTEMPTS = 3;
  
  // Refresh token handling
  async refreshSession() {
    try {
      const { data, error } = await this.supabase.auth.refreshSession();
      // Handle refresh result
    } catch (error) {
      // Implement retry logic
    }
  }

  // Auto-refresh setup
  setupAutoRefresh() {
    // 1. Regular interval check
    const intervalId = setInterval(() => this.refreshSession(), this.REFRESH_INTERVAL);
    
    // 2. Tab focus check
    const focusHandler = () => this.refreshSession();
    window.addEventListener('focus', focusHandler);
    
    // 3. Network recovery check
    const onlineHandler = () => this.refreshSession();
    window.addEventListener('online', onlineHandler);

    // Return cleanup
    return () => {
      clearInterval(intervalId);
      window.removeEventListener('focus', focusHandler);
      window.removeEventListener('online', onlineHandler);
    };
  }
}
```

### 6. SSR Profile Loading (lib/server-profile.ts)

```typescript
export async function getServerProfile(cookies: RequestCookies) {
  const supabase = getServerClient(cookies);
  
  try {
    // 1. Get user from auth
    const { data: { user } } = await supabase.auth.getUser();
    if (!user) return null;

    // 2. Get profile from database
    const { data: profile } = await supabase
      .from('users')
      .select('*')
      .eq('id', user.id)
      .single();

    return profile;
  } catch (error) {
    console.error('Failed to load server profile:', error);
    return null;
  }
}

// Usage in Server Component or Layout
export default async function RootLayout({ children }) {
  const profile = await getServerProfile(cookies());
  
  return (
    <AuthProvider initialProfile={profile}>
      {children}
    </AuthProvider>
  );
}
```

```typescript
interface AuthErrorBoundaryProps {
  level: 'root' | 'component';
  fallback?: React.ReactNode;
}

export class AuthErrorBoundary extends React.Component<AuthErrorBoundaryProps> {
  state = { error: null, errorInfo: null };

  static getDerivedStateFromError(error: Error) {
    return { error };
  }

  componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
    // Log to error reporting service
  }

  render() {
    if (this.state.error) {
      if (this.props.level === 'root') {
        return (
          <GlobalAuthError 
            error={this.state.error}
            onRetry={this.handleRetry}
          />
        );
      }
      
      return this.props.fallback || (
        <ComponentAuthError 
          error={this.state.error}
          onRetry={this.handleRetry}
        />
      );
    }

    return this.props.children;
  }
}
```

### 8. Auth Context Implementation

```typescript
export function AuthProvider({ 
  children, 
  initialProfile 
}: AuthProviderProps) {
  // Initialize with SSR data if available
  const [profile, setProfile] = useState(initialProfile);
  const sessionManager = useRef(new SessionManager());

  useEffect(() => {
    // 1. Set up session refresh
    const cleanup = sessionManager.current.setupAutoRefresh();

    // 2. Set up auth state listener
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      async (event, session) => {
        if (event === 'SIGNED_IN') {
          const profile = await fetchProfile(session.user.id);
          setProfile(profile);
        } else if (event === 'SIGNED_OUT') {
          setProfile(null);
        }
      }
    );

    // 3. Set up store revalidation
    setupRevalidation(useProfileStore);

    return () => {
      cleanup();
      subscription.unsubscribe();
    };
  }, []);

  return (
    <AuthContext.Provider value={{ profile, /* other values */ }}>
      {children}
    </AuthContext.Provider>
  );
}
```
