Implementation Plan: Lessons Booking System

Overview
Based on my analysis of the codebase, I'll implement a comprehensive Lessons Booking System that integrates seamlessly with the existing architecture. The system will follow the established patterns for court reservations, event management, and user authentication.

✅ Phase 1: Database Schema & Migrations - COMPLETE

✅ 1.1 Update Users Table
✅ Add Field: coaching_rate (decimal, nullable) - for coaches only

✅ Add Field: specialties (text[], nullable) - for coaches only
✅ Update Role: Add 'coach' as a valid role option

✅ 1.2 Create Coach Availability Table
✅ Table: coach_availability
✅ Fields: id, coach_id (FK to users), start_time, end_time, recurring_rule (JSON), created_at, updated_at, deleted_at
✅ RLS Policies: Coaches can manage their own availability, admins can manage all

✅ 1.3 Update Events Table
✅ Add Field: coach_id (FK to users table, nullable)
✅ Add Field: lesson_duration (integer, default 60 minutes)

✅ Add to event_types: "Lesson" with appropriate cost structure

✅ Add lesson pricing to membership_event_costs table for different membership types

✅ Phase 2: API Endpoints - COMPLETE

✅ 2.1 Coach Management APIs
✅ GET /api/coaches - List available coaches (users with role='coach')
✅ GET /api/coaches/[id] - Get coach details
✅ POST /api/coaches - Create coach (admin only, creates user with role='coach')
✅ PUT /api/coaches/[id] - Update coach (admin/coach only)
✅ DELETE /api/coaches/[id] - Delete coach (admin only, soft delete)

✅ 2.2 Coach Availability APIs
✅ GET /api/coaches/[id]/availability - Get coach availability
✅ POST /api/coaches/[id]/availability - Set coach availability
✅ PUT /api/coaches/[id]/availability/[availabilityId] - Update availability
✅ DELETE /api/coaches/[id]/availability/[availabilityId] - Delete availability

✅ 2.3 Lesson Booking APIs
✅ GET /api/lessons/availability - Get available lesson slots
✅ POST /api/lessons/book - Book a lesson
✅ GET /api/lessons - Get user's booked lessons (with filtering)
✅ GET /api/lessons/[id] - Get specific lesson details
✅ DELETE /api/lessons/[id] - Cancel lesson

🔄 2.4 Pricing APIs
🔄 GET /api/lessons/pricing - Get lesson pricing based on coach and membership
