## Courts API 404 Investigation & Fix

### ✅ Issue Identified
**Problem**: Mobile app getting 404 error when accessing `/api/courts`
**Root Cause**: Missing general `/api/courts` endpoint - only admin and specific endpoints existed

**Existing Courts Endpoints**:
- `/api/admin/courts` - Admin-only courts management
- `/api/courts/availability` - Court availability checking  
- `/api/courts/reserve` - Court reservation
- `/api/admin/courts/schedule` - Admin court schedule

**Missing**: General `/api/courts` endpoint for fetching courts

### ✅ Solution Implemented

**Created**: `apps/web/app/api/courts/route.ts`

**Features**:
1. **GET /api/courts** - Fetch courts with optional location filtering
2. **POST /api/courts** - Create new court (admin functionality)
3. **Proper Error Handling** - Comprehensive error responses
4. **Logging** - Added distinctive logging for debugging
5. **Location Filtering** - Optional `locationId` query parameter

**API Response Format**:
```json
{
  "courts": [
    {
      "id": "uuid",
      "name": "Court 1",
      "description": "Court description",
      "location_id": "uuid",
      "created_at": "2024-01-15T10:00:00Z",
      "updated_at": "2024-01-15T10:00:00Z"
    }
  ],
  "count": 1
}
```
