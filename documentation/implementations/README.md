## How to Use This Directory

### Starting a New Project
1. **Archive Current Project**: Move completed project details to a new file with descriptive name and date
2. **Clear Active Project**: Reset `active_project.txt` with new project template
3. **Update Cursor Rules**: Add any new learnings to `.cursorrules` file

### Project Naming Convention
- Use descriptive names that clearly indicate the project purpose
- Include completion date in format: `YYYY-MM-DD`
- Example: `mobile-app-monorepo-restructuring-2025-01-17.md`

### File Organization
- **Completed Projects**: Move to archived files with descriptive names
- **Active Project**: Keep current status in `active_project.txt`
- **Learnings**: Update `.cursorrules` with new best practices
- **Documentation**: Keep all technical details and implementation notes

## Best Practices

### Documentation
- **ALWAYS** document all major changes and fixes
- **ALWAYS** include file paths and line numbers when relevant
- **ALWAYS** document authentication patterns and security measures
- **ALWAYS** keep track of deployment procedures

### Code Quality
- **ALWAYS** run linting before committing
- **ALWAYS** test functionality before deploying
- **ALWAYS** update documentation when making changes
- **ALWAYS** follow established patterns in the codebase

### Security
- **ALWAYS** implement RLS policies before exposing new tables
- **ALWAYS** use proper authentication patterns
- **NEVER** expose sensitive operations to client-side
- **ALWAYS** validate user permissions before database operations
